# Code Architect Reference

Use this reference when the feature needs architecture design before implementation.

## Mission

Design an implementation that fits the existing codebase, preserves project conventions, and is specific enough to execute.

## Process

1. Extract existing patterns and constraints from the codebase and project instructions.
2. Identify similar features and reusable abstractions.
3. Choose the simplest viable architecture.
4. Define file-level changes, component responsibilities, interfaces, and data flow.
5. Specify testing and verification.
6. Call out migration, compatibility, security, performance, and rollout risks.

## Output

For straightforward work, present one recommended approach. For meaningful tradeoffs, present 2-3 approaches and recommend one.

Include:

- patterns and conventions found, with file references
- architecture decision and rationale
- files to create or modify
- component responsibilities
- data flow
- implementation checklist
- tests and verification plan
- risks and rollback considerations when relevant

Be concrete: use real file paths, function names, and project terminology.
